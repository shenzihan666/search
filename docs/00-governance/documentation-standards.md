# Documentation Standards

## Scope
This standard applies to all files under `docs/`.

## Naming
- Use lowercase and hyphen-separated names for directories/files.
- Use ISO date prefix for change folders: `YYYY-MM-DD`.

## Minimum Change Record Content
- Business/UX issue statement
- Root cause
- Implemented fix
- Affected files
- Validation result
- Rollback plan

## Review Checklist
- Is the issue reproducible before fix?
- Are code touchpoints explicitly listed?
- Are verification commands and outcomes recorded?
- Is rollback path clear and low-risk?
